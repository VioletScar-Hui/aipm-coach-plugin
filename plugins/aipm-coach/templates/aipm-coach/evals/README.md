# 评测集（evals）—— 量"系统抓不抓得住关键问题"

> 固定的、人工构造的复盘产出样本（不是真实面试）。每个样本故意藏了一个该被抓的坑，或反过来是合格样本不该被乱报。
> 跑 `/aipm-coach:aipm-eval` 模式 B：让全新上下文的检查者对每个样本独立打分，再和文末「期望评判」对比，报命中 / 漏判 / 误报。

## 怎么用
- 说"跑评测" → 跑全部 `case-*.md`。
- 看稳定性：要求跑 3 次，看判定是否一致（`pass^k`——agent 要可复现，不是偶尔灵光）。

## 正负均衡
- 负样本（case-01 编造、case-02 答非所问）：测系统**抓得住**。
- 正样本（case-03）：测系统**不乱报**。只有负样本会纵容一个"把一切都判不合格"的偷懒检查者——正样本是它的照妖镜。

## 改进闭环
真实复盘里发现检查者**漏判**了一类新坑 → 把那段固化成新的 `case-*.md`（带期望评判）→ 以后每次都测它，并回去给 `aipm-retro` 的 Verify 加一条对应自检。评测集越攒越全，系统越来越准。

> 提示：建议把这两个负样本换 / 补成**你自己最常栽的坑**（见 `weak-spots.md`），系统对你才最有针对性。
