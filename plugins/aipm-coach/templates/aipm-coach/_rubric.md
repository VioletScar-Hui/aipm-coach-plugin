# AIPM 面试回答评估标准（Rubric）

> 这是诊断你回答、生成优质答案、提炼话术时的**统一度量衡**。
> 它被 `aipm-retro` / `aipm-resume` / `aipm-prep` 三个 skill 按需读取。
> 你可以随时修改本文件来校准标准（比如某家公司更看重某一维度）。

## 一、十维评分（每维 优 / 中 / 差，必须给出具体证据）

### 通用 PM 维度（5）

| # | 维度 | 优（满分） | 差（扣分） |
|---|------|-----------|-----------|
| 1 | **结构性** | 结论先行；用 STAR / 总分总 / 假设-验证等框架组织 | 流水账、想到哪说到哪、听完不知道重点 |
| 2 | **量化结果 / impact** | 有可量化结果（指标变化、规模、收益）且归因清晰 | 只讲"做了什么"，没有结果或结果无法量化 |
| 3 | **用户 / 场景洞察** | 能讲清"为谁、解决什么真实问题、为何是痛点" | 从功能出发、不谈用户；对场景理解模糊 |
| 4 | **决策与权衡** | 展示了优先级、取舍、为什么不做 B 选 A | "我什么都做了"、没有取舍、没有决策逻辑 |
| 5 | **表达** | 简洁、抓重点、可被打断后能收回主线 | 冗长、绕、术语堆砌、答非所问 |

### AI 特有维度（5）—— AIPM 与普通 PM 的分水岭

| # | 维度 | 优（满分） | 差（扣分） |
|---|------|-----------|-----------|
| 6 | **模型能力与边界** | 懂模型能做/不能做（幻觉、上下文、确定性），知道 prompt / RAG / 微调 / Agent 各自适用场景，但不假装是工程师 | 把 LLM 当万能锤；或完全不懂技术只谈情怀；或炫技式堆名词 |
| 7 | **AI 产品评估体系** | 能设计衡量一个 AI 功能好坏的方法：离线指标（准确率/召回）+ 幻觉率 + 人工评测 + 在线 A/B + 用户满意度，并知道何时用哪种 | 只说"看效果好不好"；没有评估框架；混淆模型指标与产品指标 |
| 8 | **数据飞轮与迭代** | 说得清数据如何回流、如何驱动下一轮提升、冷启动怎么办 | 不谈数据闭环；把 AI 产品当一次性交付 |
| 9 | **AI 风险与边界** | 主动谈幻觉/安全/合规/隐私，以及成本-延迟-质量的三角权衡 | 完全不提风险与成本；或只会喊"要注意安全"无具体手段 |
| 10 | **AI 商业判断** | 能判断"这个场景到底值不值得用 AI"、ROI、替代方案 | 为了 AI 而 AI；不算账；分不清 demo 与可商业化 |

## 二、常见扣分项（Anti-patterns，诊断时优先扫描）

- 流水账：没有框架，听完抓不到重点
- 无量化：通篇没有一个数字 / 规模 / 结果
- 无取舍：把自己讲成"全能选手"，没有任何决策与放弃
- 答非所问：没听清问题真正想考的点
- 套话空话：「我很有 owner 意识」却没有事实支撑
- 角色错位：把自己讲成工程师（炫技）或运营（只谈执行），不像产品决策者
- AI 万能锤：任何问题都答"用大模型解决"，不算成本、不谈边界
- 评估缺失：讲了一个 AI 功能，却答不出"你怎么知道它做得好"
- 简历不一致：口述与简历对不上、数字前后矛盾

## 三、题型分类（用于 question-bank 归档与 prep 抽题）

`SELF` 自我介绍 / 职业动机 ·
`PROJ` 项目深挖（最有成就感 / 最难 / 失败的项目）·
`DESIGN` 产品设计（设计一个 AI 功能 / 改进某产品）·
`TECH` AI 技术理解（RAG / Agent / 微调 / 评估 / 幻觉）·
`METRIC` 指标与北极星 / 如何衡量成功 ·
`BQ` 行为面（冲突 / 失败 / 影响他人 / 领导力）·
`CASE` case / 估算 / 分析 ·
`WHY` 为什么 AI / 为什么我们 / 为什么你 ·
`REVQ` 反问环节

## 四、优质答案的硬约束（生成范例时必须遵守）

1. **真实可用**：答案必须建立在你简历 / star-stories 里的真实经历上；缺事实时用 `[需你补充：xxx]` 占位，**绝不编造数字或项目**。
2. **结论先行**：第一句给结论 / 框架，再展开。
3. **可量化**：尽量带上指标或规模；缺失则标 `[需你补充：量化数据]`。
4. **AI 含量**：AIPM 答案要体现至少 1 个 AI 特有维度（评估 / 数据飞轮 / 风险 / 边界）。
5. **电梯版 + 完整版**：每个优质答案给「30 秒电梯话术」+「完整版」两档。
