# 进阶诊断方法（_diagnosis-plus）

> retro / prep / eval 按需读取的"深度剖析工具箱"。配合 `_rubric.md`（基础十维）使用——rubric 判"好不好"，本文件判"为什么、差多远、怎么追问、可不可信"。
> 这是把复盘从"打分"升级为"教学"的关键。

## ④ 四层归因（每条诊断/弱点必须标一层——不同层不同疗法）
诊断时，除了给 rubric 维度，再判它属于哪一层（决定怎么治）：

| 层 | 是什么 | 疗法 | 典型表现 |
|---|---|---|---|
| **表达层** | 说法/结构/长度/口头禅——知道但没说好 | **重练 + 口述体检**（见 drills），最快见效 | 不结论先行、答得太长 |
| **思维层** | 判断/取舍/框架缺失——想得不够 | **重答 + 追问树**反复打，建立框架 | 答非所问、缺权衡 |
| **知识层** | 真的不懂的东西 | **补课 + 专项练**，不能靠临场 | 某领域建模/概念不熟 |
| **心态层** | 过度包装/讨好/回避/紧张 | **认知矫正**——讲清代价（被追问击穿） | 埋没强项、夸大弱项 |

> 在 `weak-spots.md` 给每条弱点打上 `层=表达/思维/知识/心态`。prep 抽题时：知识层优先补课，表达层优先口述练，思维层优先追问树。

## ⑤ 追问树 + 击穿点预测（真面试都死在连环追问）
对每道**核心题/答得弱的题**，生成 3 层追问，并预测用户会在第几问崩：
- **第 1 问（澄清）**：你这个具体指什么？举个例？
- **第 2 问（rationale）**：**为什么**这么做/这个顺序？凭什么？——*高发崩点：把"为什么"答成"好处"。*
- **第 3 问（边界/数据）**：边界在哪？数据/信息从哪来？失败/异常怎么办？规模多大？
每层标 ✅能扛 / ⚠️勉强 / 🔴会崩，并对 🔴 给"补法 + 一句应答模板"。

## ⑥ 距满分的具体差距（把模糊评价变成可勾待办）
"更优解"不要写"答得一般、再深入些"。写成**距 rubric 的'优'还差的具体动作清单**：
- [ ] 缺一个候选方案对比（A/B/C 为什么不选）
- [ ] 缺一个收口数字 / 指标
- [ ] 缺一个 AI 特有维度（评估/风险/数据飞轮）
- [ ] 首句不是结论
每条都是能当场改的动作，不是感受。

## ⑦ 面试官意图解码（教读潜台词，而非答字面）
每道题标一行"**他真正在考什么**"：
- "讲个项目" → 考你是不是产品决策者（取舍）还是执行者（罗列）。
- "为什么这个顺序" → 考设计 rationale，不是收益。
- "能一直实习吗"（连问） → 隐性硬门槛/否决项，不是闲聊。
- "你怎么衡量它好不好" → 考 AI 评估体系（AIPM 必考）。
让用户学会"先想他在考什么，再答"。

## ⑩ 可信度审计（进考场前把自己的 claim 压一遍）
扫用户答案 / 简历里**经不起追问的点**，逐条给"被追问怎么交代"：
- **夸大用词**：生产级 / 海量 / 大幅 / 完美——是否名实相符？
- **交代不清的数字**：某指标怎么测的、什么窗口、归因是不是你？任一数字都要想好口径。
- **含糊归属**："我做了 X" 里你到底负责哪部分？被追问"具体你干的"答得出吗？
- **借来的数据**：引用的是公开/行业数还是自己做的？要主动说明（别让面试官以为是你的战绩）。
输出："这 3 个 claim 有风险，被追问的标准答法是……"
